## 2. 일시·장소 표기 (`74b193e` / `a323a3e` 2026-06-23)

`fix(demoday): 히어로 일시 표기를 KST로 강제` · `fix(admin/demoday): 회차 편집 시각을 KST wall-clock으로 처리`

- 증상: 히어로의 `WHEN`이 14:00–18:00 KST가 아니라 05:00–09:00으로 표시.

  시:분을 모두 한국 시간으로 강제. 마케팅 페이지 + 어드민 input 초깃값/
  서버 액션 모두 동일 가정으로 통일 (`+09:00` suffix를 붙여 해석).
- 컬럼 `event_end_date` 추가 (마이그레이션 `0005`).

## 5. 어드민 로그인 무한 리다이렉트 (Supabase Auth 설정)

- 증상: `/admin` → Google 로그인 진행했지만 대시보드 진입 실패, 다시
  `/admin/login`으로 회귀 무한 루프.
- 진단 (Phase 1 증거 수집):
  - Vercel 런타임 로그: `/admin/auth/callback` 호출이 0건.
  - Supabase Auth 로그: `/authorize` 302 → Google `/callback` 302 →
    `login` 성공 (`yonseivery1997@gmail.com`).
  - 즉 Supabase는 인증을 끝냈는데 우리 콜백 URL로 사용자를 못 보냄.
- 원인: Supabase **Auth → URL Configuration**의 Redirect URLs
  화이트리스트에 `https://yonseivery.com/admin/auth/callback` 미등록.
  허용 목록 없을 시 `redirectTo`가 거부되고 Site URL(보통 dev)로
  fallback되어 사용자가 localhost로 이탈.
- 조치: 사용자가 Site URL을 production 도메인으로, Redirect URLs에
  `https://yonseivery.com/admin/auth/callback`,
  `https://www.yonseivery.com/admin/auth/callback`,
  `http://localhost:3000/admin/auth/callback`을 추가하고 저장.
- 참고: `is_admin` RPC + `public.admins` 화이트리스트
  (`ricky7@yonsei.ac.kr`, `yonseivery1997@gmail.com`) 자체는 정상이었다.

## 6. 포스터 업로드 400 (`9c99021` 2026-06-23)

`fix(admin/demoday): 포스터 업로드 body 1MB 한도 초과로 인한 400`

- 증상: 어드민에서 포스터 파일 선택 후 "업로드" 클릭 → 오류.
- 진단: Vercel 런타임 로그에서 `POST /admin/demoday/<id> 400`,
  메시지 `"Body exceeded 1mb limit"`. Next.js Server Action 기본
  body 한도(1MB)에 막혀 server action 자체에 도달하지 못함.
- 조치: `next.config.ts`에
  `experimental.serverActions.bodySizeLimit: '6mb'` 추가
  (`uploadDemodayPoster`의 5MB 검증을 통과시키고 multipart overhead
  여유). Storage bucket/정책, MIME/사이즈 검증은 그대로 유지.
